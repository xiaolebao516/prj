# 软件架构与核心模块分析报告 (Open Image System)

## 1. 宏观架构提取 (Architecture)

### 1.1 系统类型
- **类型**：裸机 (Bare-metal) 程序
- **理由**：代码中没有 FreeRTOS 相关的头文件或 API 调用，主函数采用无限循环轮询方式处理任务

### 1.2 工程目录结构
系统严格遵循了标准嵌入式分层架构，归类如下：

| 层级 | 目录/文件 | 职责 |
|------|-----------|------|
| **APP（业务控制）** | `oiMain.c`, `oiShotMan.c`, `oiSm.c` | 主程序逻辑、超声拍摄管理、状态机 |
| **Middleware（中间件）** | `oiServer.c`, `oiCmd.c` | LwIP网络通信、上位机命令解析 |
| **BSP（板级支持）** | `platform.h`, `platform_config.h`, `platform_zynqmp.c` | 平台级初始化、时钟与缓存配置 |
| **HAL（硬件抽象）** | `oiAdc.c`, `oiAdcDma.c`, `oiPulser.c`, `oiTgc.c` | AD9670/高压脉冲器/TGC等外设的寄存器级控制 |

## 2. 核心外设追踪 (Core Peripherals)

### 2.1 AD9670 寄存器配置 (模拟前端)
**核心文件**：`oiAdc.c`
**初始化流程**：
1. **SPI 初始化**：通过 `XSpiPs_LookupConfig` 和 `XSpiPs_CfgInitialize` 挂载 SPI 外设。
2. **芯片握手**：读取 `AD9670_REG_CHIP_ID` 寄存器，确认芯片存活及 SPI 通信正常。
3. **海量寄存器烧录**：
   - 写入基础寄存器表（`AD9670_REGS`）。
   - 写入 ADC 专属配置（`ADC_SPECIFIC_REGS`），包括：通道分配、采样率、数字解调、JESD204B 串行格式等。
4. **硬件使能**：拉高 GPIO 引脚 `EMIO_GPIO_PIN_ADC_ENABLE`，正式启动 ADC。

### 2.2 DMA 数据流 (海量超声数据搬运)
**核心文件**：`oiAdcDma.c`
**工作机制**：
1. **DMA 装配**：通过 `XAxiDma_CfgInitialize` 启动 AXI DMA 控制器，采用 Scatter-Gather (散列采集) 模式。
2. **内存映射**：使用硬件级写死的物理内存地址 `SAMPLE_BUFFER_ADDRESS` 作为超声回波数据的落盘锚点。
3. **轮询监控**：在 `oiAdcDmaVisit()` 中不断查询 DMA 状态描述符，一旦侦测到传输完成，立即抛出 `EVENT_SHOT_DONE` 事件给上层状态机。

### 2.3 高速网络传输 (回传上位机)
**核心文件**：`oiServer.c`
**工作机制**：
1. **协议栈拉起**：调用 `lwip_init()` 初始化 LwIP，绑定网卡 MAC 和静态 IP。
2. **TCP 服务端**：监听指定的 `OI_TCP_PORT`，等待上位机（如 Python 或 C# 客户端）接入。
3. **指令与数据拆分**：通过 `recv_callback()` 接收上位机的拍摄参数；拍摄完成后，应对大数据量限制，将内存中的超声数据分片打包，通过 `oiServerReply()` 狂飙发回上位机。

## 3. 程序的启动与流转 (Main Execution Flow)

### 3.1 上电初始化顺序 (Init Sequence)
1. `main()` 启动，进入 `oiInit()`。
2. 基础底盘拉起：关闭 Cache 防止 DMA 数据不一致 (`Xil_DCacheDisable`)，初始化 GPIO。
3. 外设逐个点名：依次调用 `oiAdcInit()`, `oiAdcDmaInit()`, `oiPulserInit()`, `oiServerInit()`, `oiShotManInit()`。
4. 状态机切入：初始化完毕，发送 `EVENT_INIT_COMPLETE`，系统进入待命 `STATE_READY`。

### 3.2 核心主循环 (The Super Loop)
这是一个极其典型的非阻塞状态机轮询架构：
```c
while(1) {
    oiAdcVisit();      // 巡视 ADC 状态
    oiAdcDmaVisit();   // 巡视 DMA 是否搬运完成
    oiPulserVisit();   // 巡视 高压脉冲发射器
    oiServerVisit();   // 处理 LwIP 网络收发请求
    oiShotManVisit();  // 拍摄流程总管家
    oiSmVisit();       // 状态机核心推演
}
